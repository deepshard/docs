## Prerequisites

- Python 3.12 or higher
- A Truffle¹ device
- The [Symphony desktop client](/client/overview), with your device already onboarded

## Install Python

<Tabs>
  <Tab title="macOS">
    ```bash
    brew install python@3.12
    ```

    Verify:
    ```bash
    python3 --version
    ```
  </Tab>
  <Tab title="Windows">
    Download and install Python from [python.org](https://www.python.org/downloads/), or use Chocolatey:

    ```powershell
    choco install python --version=3.12.0
    ```

    Verify:
    ```powershell
    python --version
    ```
  </Tab>
  <Tab title="Linux">
    ```bash
    sudo apt update
    sudo apt install python3.12 python3.12-venv python3-pip
    ```

    Verify:
    ```bash
    python3 --version
    ```
  </Tab>
</Tabs>

## Set up a virtual environment

<Tabs>
  <Tab title="venv">
    ```bash
    python3 -m venv .venv
    ```

    <Tabs>
      <Tab title="macOS / Linux">
        ```bash
        source .venv/bin/activate
        ```
      </Tab>
      <Tab title="Windows">
        ```powershell
        .venv\Scripts\Activate.ps1
        ```
      </Tab>
    </Tabs>
  </Tab>
  <Tab title="uv">
    If you prefer [uv](https://github.com/astral-sh/uv) for faster package management:

    ```bash
    # install uv if you don't have it
    curl -LsSf https://astral.sh/uv/install.sh | sh

    # create and activate a venv
    uv venv
    source .venv/bin/activate   # macOS / Linux
    # or
    .venv\Scripts\Activate.ps1  # Windows
    ```
  </Tab>
</Tabs>

## Install Truffile

With your environment activated:

```bash
pip install truffile
```

Verify the installation:

```bash
truffile help
```

You should see the Truffile welcome panel listing every command (`scan`, `connect`, `deploy`, `chat`, `infer`, and the rest). If you see the panel, you're ready to go.

<Note>
Make sure you've onboarded your Truffle with the Symphony desktop client and grabbed your User ID from **Settings → About** before running `truffile connect`.
</Note>

## Agent setup prompt

If you want Codex, Claude Code, or another coding agent to set up Truffile for you, paste this prompt into the agent from the directory where you want the environment created:

```text
Set up Truffile for me in this directory.

Use explicit CLI commands and explain what you are doing as you go.

Requirements:
- Make sure Python 3.12 or newer is available. Prefer `python3.12` when present; otherwise use the newest `python3`/`python` that is >=3.12.
- Prefer `uv` if it is installed. If `uv` is not installed, use the standard library `venv` module.
- Create a local virtual environment named `.venv`.
- Activate/use that virtual environment for all Python and pip commands.
- Install Truffile with `pip install truffile`.
- Verify the install with `truffile --help`.
- Do not require a Truffle device to finish setup. If no device is connected yet, tell me how to run `truffile scan` and `truffile connect <device-name> --user-id <user-id>` when I am ready. My user ID is available in Symphony Settings > About.

After installing, inspect the installed Truffile package and tell me where these bundled resources live locally:
- bundled skills under `truffile/skills`
- bundled example apps under `truffile/app-store`

You can locate them with Python like this:

python - <<'PY'
from importlib import resources
root = resources.files("truffile")
print("skills:", root / "skills")
print("example apps:", root / "app-store")
PY

If you support skills or instruction files, read the relevant bundled Truffile skills before using the CLI:
- `truffile-cli` for scan/connect/create/validate/deploy/list/delete/obsidian
- `truffile-chat` for talking to the on-device agent
- `truffile-infer` for raw model inference and MCP testing
- `truffle-app-creator` for creating or porting Truffle apps

When setup is complete, give me a short onboarding that covers:
- how to connect a Truffle device
- how to chat with the on-device agent
- how to use Truffile as an inference endpoint/client
- how to create, validate, deploy, list, and delete apps
- where the example apps live inside the installed package
- links to the docs:
  - https://docs.truffle.net/sdk/installation
  - https://docs.truffle.net/sdk/cli
  - https://docs.truffle.net/sdk/building-apps
```

The prompt works because Truffile is CLI-first: every core workflow has explicit arguments, and the package ships with both agent-readable skills and example apps.
