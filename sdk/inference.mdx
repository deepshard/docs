Your Truffle exposes inference through three SDK commands:

- `truffile models` — list models on your connected device
- `truffile chat` — interactive chat REPL (runtime config is done with `/` commands)
- `truffile proxy` — local OpenAI-compatible proxy

---

## Quick Start

### 1) List models

```bash
truffile models
```

This calls the device endpoint:

- `GET /if2/v1/models`

---

### 2) Start chat REPL

```bash
truffile chat
```

`truffile chat` is REPL-first. You do not need startup flags for tuning.  
Use slash commands inside chat to configure behavior live.
Use `truffile chat` by itself (chat positional args are not used).

Chat requests are sent to:

- `POST /if2/v1/chat/completions`

On startup you will see:

- resolved default model
- connected device
- available REPL commands

---

### 3) Start OpenAI-compatible proxy

```bash
truffile proxy
```

By default:

- listen: `http://127.0.0.1:8080`
- OpenAI base URL: `http://127.0.0.1:8080/v1`
- upstream device API: `/if2/v1/*`
- you can use something like ngrok to forward this port as well!

## Chat Commands

Use `/help` in REPL to list commands.

### Core REPL commands

| Command | What it does |
|---------|---------------|
| `/help` or `/` | Show available commands |
| `/history` | Show current conversation history summary |
| `/reset` | Clear current conversation state |
| `/models` | Open model picker and switch active model |
| `/config` | Show current chat config values |
| `/exit` or `/quit` | Exit chat |

### Generation controls

| Command | What it does |
|---------|---------------|
| `/reasoning on\|off` | Toggle reasoning output |
| `/stream on\|off` | Toggle streaming output |
| `/json on\|off` | Toggle JSON response mode |
| `/tools on\|off` | Toggle built-in default tools (`web_search`, `web_fetch`) |
| `/max_tokens <int>` | Set max output tokens |
| `/temperature <float\|off>` | Set/clear temperature |
| `/top_p <float\|off>` | Set/clear top-p |
| `/max_rounds <int>` | Set max assistant/tool loop rounds |
| `/system <text>` | Set system prompt |
| `/system clear` | Clear system prompt |

### MCP in chat

| Command | What it does |
|---------|---------------|
| `/mcp status` | Show connected MCP endpoint + tool count |
| `/mcp connect <streamable-http-url>` | Connect external MCP server |
| `/mcp tools` | List tools from connected MCP |
| `/mcp disconnect` | Disconnect MCP session |

`/mcp connect` expects an HTTP(S) streamable MCP endpoint.

## MCP Testing Workflow (Local and Hosted)

`truffile chat` can be used as an MCP integration test harness before app deployment.

### Local MCP test

1. Start your local MCP server with streamable HTTP transport.
2. In chat, connect it:
   - `/mcp connect http://127.0.0.1:8000/mcp`
3. Verify discovered tools:
   - `/mcp tools`
4. Ask for explicit tool use and verify calls in output.

### Hosted MCP test (Exa example)

Use a hosted MCP endpoint exactly the same way:

- `/mcp connect https://mcp.exa.ai/mcp?exaApiKey=<EXA_API_KEY>&tools=web_search_exa,web_search_advanced_exa,get_code_context_exa,crawling_exa,company_research_exa,people_search_exa,deep_researcher_start,deep_researcher_check`
- `/mcp tools`
- prompt example: `use the company research exa tool to find me information about SpaceX`

## Proxy Behavior

`truffile proxy` exposes:

- `GET /v1/models`
- `POST /v1/chat/completions`

Supported options:

- `--device <name>`: use a specific connected device
- `--host <host>`: bind interface (default `127.0.0.1`)
- `--port <port>`: bind port (default `8080`)
- `--no-think-tags`: keep reasoning separate (`reasoning_content`) instead of injecting `<think>...</think>` into content

## OpenAI SDK Example

```python
from openai import OpenAI

client = OpenAI(
    base_url="http://127.0.0.1:8080/v1",
    api_key="not-required",
)

resp = client.chat.completions.create(
    model="<model-id-or-uuid-from-truffile-models>",
    messages=[{"role": "user", "content": "Give me one sentence about Truffle."}],
)

print(resp.choices[0].message.content)
```

## Command Reference

| Command | Purpose |
|---------|---------|
| `truffile models` | List models on connected Truffle |
| `truffile chat` | Start interactive chat REPL |
| `truffile proxy` | Start local OpenAI-compatible proxy |
| `truffile proxy --port <port>` | Run proxy on custom port |
| `truffile proxy --host <host>` | Bind proxy to specific interface |
| `truffile proxy --device <name>` | Proxy against specific Truffle device |
| `truffile proxy --no-think-tags` | Disable `<think>` tag injection in proxy output |
